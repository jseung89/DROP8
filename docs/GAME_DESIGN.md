# 게임 디자인

## 핵심 루프

로비 → 비행기 → 낙하 → 맨손 파밍 → 무기 획득 → 자기장 이동과 교전 → 최후의 생존자 → 결과 및 리매치

## 맵

논리 크기 4096×4096이며 폐공장, 주택가, 병원, 창고지대, 군사시설, 숲속 캠프를 포함합니다. 장애물은 서버와 클라이언트가 공유하는 직사각형 데이터로 관리합니다.

## 장비

- 기본: 주먹
- 보조: 권총
- 주무기: 기관단총, 돌격소총, 샷건, 저격소총
- 근접: 야구방망이, 프라이팬, 쇠파이프, 식칼
- 탄약: 소형탄, 소총탄, 산탄
- 투척: 파편 수류탄, 연막탄, 화염탄
- 생존: 방탄조끼, 붕대, 구급상자

저격소총은 방향형 스코프와 이동 제약을 적용해 서버 권위로 판정합니다. 오토바이에서는 정지에 가까운 상태에서만 스코프를 사용할 수 있습니다.

## AI

AI는 비행기에서 분산 낙하하고, 총이 없으면 무기를 탐색하며, 시야선이 확보된 적과 교전하고 자기장 밖이면 중앙을 향해 이동합니다. 난이도는 이동속도와 판단 주기에 영향을 주며 체력과 무기 피해를 부당하게 높이지 않습니다.

AI는 건물 안팎 목표가 다르면 문 또는 넘을 수 있는 창문을 중간 경유점으로 선택합니다. 창문은 서버 권위 vault 상태를 사용하며 벽 통과나 순간이동으로 대체하지 않습니다. 장애물 고착 시 제한 주기로 경로를 다시 계산하고 여러 각도의 지역 이동 후보를 비교합니다. 연막은 시야만 차단하고, 화염은 높은 이동 비용으로 평가합니다.

## 채팅

- 로비: 방 전체
- 생존 중: 거리 제한 근거리 채팅
- 사망 후: 관전자 전용 채팅
- 빠른 문구 6종

## Refactor 013 공간과 수영 규칙

### 실내 공간
- 외부는 `roomIndex=0`, 건물 내부는 맵별 고유한 숫자 roomIndex를 사용합니다.
- 같은 방은 일반 시야를 사용하고 다른 방은 실제 문 또는 창문 opening을 통과하는 선분만 보입니다.
- 막힌 방의 구조는 흐리게 남기되 플레이어, AI, 전리품, 이름표와 미니맵 정보는 숨깁니다.
- 문은 이번 버전에서 항상 열린 통로이며 내부 창문은 플레이어와 AI가 동일한 서버 권위 vault를 사용합니다.

### 8번 부두
- 크기: 7168×7168
- 강: 중심선 기반 굴곡형, 일반 폭 720~900px, 최대 1040px
- 횡단: 차량 다리 1, 관리 다리 1, 보행자 다리 1, 여울 2
- 전리품: 264개 후보 중 216개 활성, 실내 124개(약 57.4%), 깊은 물 생성 금지
- 오토바이: 12개 후보 중 8대 활성, 강 양쪽과 중앙 접근로 분산

### 수영
- 깊은 물 수영 속도는 165px/s입니다.
- 수영 중 총기·근접·재장전·투척·회복·습득·차량 탑승·창문 넘기는 금지됩니다.
- 무기 선택과 채팅은 허용됩니다.
- 익사, 스태미나, 잠수, 수중 총기는 포함하지 않습니다.


## Refactor 013G 8번 부두 품질 규칙

- 현재 방은 오버레이 없이 100% 표시하며, 다른 방은 포털 연결 여부에 따라 약하게 또는 강하게 가립니다. 엔티티는 공용 시야 결과로 개별 표시합니다.

- 오토바이는 서쪽 3~4대, 동쪽 3~4대, 중앙 접근로 최소 1대를 보장하며 정비소 집중을 제한합니다.
- 물가 모래·얕은 물·깊은 물과 상륙 지점을 서로 다른 색과 표식으로 표현합니다.
